## 起動方法

フロントエンド単体で試す場合:

```sh
cd frontend
docker compose up -d --build
```

起動後、ブラウザで `http://localhost/` にアクセスします。

APIバックエンド（FastAPI）も同時に動かす場合は、プロジェクトルートの `docker-compose.yml` を使います。ただし現状、ルート側・`frontend/` 側の両方の nginx がホストの **80番ポートを使う設定** になっているため、**同時に起動するとポートが衝突します**。両方を同時に立ち上げたい場合は、どちらかの `docker-compose.yml` でポート番号を変更してください（例: フロントエンド側を `8080:80` に変更）。

### 主要な環境変数（`frontend/.env`）

| 変数名 | 内容 |
|---|---|
| `SECRET_KEY` | Djangoのシークレットキー |
| `DEBUG` | デバッグモード（`True`/`False`） |
| `ALLOWED_HOSTS` | 許可するホスト名 |
| `API_ENDPOINT` | 接続先のFastAPIバックエンドのURL |
| `PRIVATE_KEY` | バックエンドとJWTを検証するための共有シークレット（バックエンドの`SECRET_KEY`と同じ値） |
| `ALGORITHM` | JWTの署名アルゴリズム（`HS256`） |

## アカウント登録・ログイン

トップページ（`/`）から「はじめる」または「新規登録」を選ぶと登録画面（`/signin/`）に進みます。

- **メールアドレス＋パスワードで登録**: フォームに入力して「登録する」を押すと、そのまま自動的にログインしてダッシュボードに移動します。
- **Googleで登録/ログイン**: 「Googleでログイン」ボタンから、Googleアカウントでの認証・カレンダー連携をまとめて行えます（設定が必要です。[Googleカレンダー連携](#googleカレンダー連携)を参照）。

すでにアカウントをお持ちの場合はログイン画面（`/login/`）からメールアドレス／パスワード、またはGoogleでログインしてください。

ログアウトはヘッダー右上の「ログアウト」から行えます。

---

## ダッシュボード

ログインすると `/dashboard/` に移動し、登録済みの予定が一覧（カード形式）で表示されます。各カードには以下が表示されます。

- 予定名
- 開始日〜終了日
- 開始時刻〜終了時刻
- 場所
- タグ

カードをクリックすると、その予定の詳細画面に移動します。

## 予定の追加（自然文入力）

ダッシュボード上部の入力欄に、日付・時刻・場所などを含む自然な文章を書いて「解析する」を押すと、内容を自動で読み取り、予定の下書き（確認画面）を作成します。

**入力例:**

```
会議 2026/08/15 10:00 11:00 東京都渋谷区 通知あり #仕事
```

**認識される要素:**

| 要素 | 書き方の例 |
|---|---|
| 日付（開始・終了） | `2026/08/15`（1つなら開始日のみ、2つなら開始日・終了日） |
| 時刻（開始・終了） | `10:00`（1つなら開始時刻のみ、2つなら開始・終了時刻） |
| タグ | `#仕事` のように半角英数字＋`#` |
| 通知 | 「通知あり」「アラーム付き」などの語句 |
| 繰り返し | 「毎日」「毎週」「毎月」「毎年」「平日」など |
| 場所 | 都道府県から始まる住所 |
| URL | `http://` または `https://` から始まる文字列 |

解析結果は確認画面に自動入力されますが、内容はすべて編集可能です。確認・修正のうえ「この内容で登録」を押すと予定が登録されます。

文章を使わず手動で予定を作りたい場合は、「手動で追加」から空の入力フォームを開けます。

---

## 予定の詳細・編集・削除

予定の詳細画面には以下のボタンがあります。

- **編集する**: 予定の内容を変更し、「更新する」で保存します。
- **削除する**: 確認画面が表示され、「削除する」を押すと完全に削除されます（取り消せません）。
- **一覧に戻る**: ダッシュボードに戻ります。

## Googleカレンダー連携

Google連携済みのアカウントでは、Task Kyleに登録した予定が自動でGoogleカレンダーにも反映され、住所をもとにした出発時刻の目安（移動時間の逆算）も計算されます。

Google連携を有効にするには、バックエンド側の設定が必要です。

1. Google Cloud ConsoleのOAuthクライアントに、**「承認済みのリダイレクトURI」** としてフロントエンドのコールバックURL（例: `http://<アクセスするホスト>/oauth/callback/`）を登録する
2. バックエンドの `.env` の `ENDPOINT` を、上記と同じURLに設定する
3. `CLIENT_ID` / `CLIENT_SECRET` がGoogle Cloud Consoleのものと一致していることを確認する

この設定が済んでいないと、「Googleでログイン」を押した際にGoogle側で `redirect_uri_mismatch` エラーになります。

## トラブルシューティング

| 症状 | 主な原因 |
|---|---|
| ページが真っ白／`Not Found` と表示される | 存在しないURLにアクセスしています。ヘッダーやリンクからの遷移か確認してください |
| デザイン（CSS）が反映されない | 静的ファイルが正しく配信されていません。`docker compose up --build` でイメージを再ビルドしてください |
| ログイン・新規登録が失敗する | `API_ENDPOINT` の設定、およびバックエンド（FastAPI）が起動しているか確認してください |
| 「Googleでログイン」で `redirect_uri_mismatch` | 上記[Googleカレンダー連携](#googleカレンダー連携)の設定を確認してください |
| フロントエンドとバックエンドを同時に起動できない | 両方のnginxが80番ポートを使っています。片方のポート番号を変更してください |
